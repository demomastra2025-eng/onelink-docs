# Справочник API One Link Cloud

One Link Cloud использует несколько API-поверхностей, потому что разные интеграционные сценарии требуют разных моделей доверия и доступа.

Основное правило выбора:

- `Application API` для операций внутри workspace
- `Client API` для клиентских чат-интерфейсов
- `Platform API` для provisioning и административных сценариев

One Link Cloud использует несколько API-поверхностей, потому что разные сценарии интеграции требуют разных моделей доверия и доступа.

## API Семейства

| Семейство API | Лучшее для | Модель аутентификации |
| --- | --- | --- |
| Application APIs | Операции внутри workspace, CRM, коммуникации, расписание, автоматизации, Captain | Токен доступа пользователя |
| Client APIs | Пользовательские чаты и клиентские messaging-приложения | Идентификатор inbox и идентификатор контакта |
| Platform APIs | Предоставление ресурсов, управляемое облачное администрирование и контроль в масштабе установки | Токен доступа на уровне платформы |

## Application APIs

Application API — основной выбор, когда вам нужно работать с One Link Cloud в качестве оператора или системы, действующей внутри workspace.

Используйте их для:

- контакты и диалоги
- inbox и командные операции
- записи CRM
- автоматизации и macros
- интеграции
- отчеты и настройки на уровне аккаунта

## Client APIs

Client API предназначены для создания пользовательских возможностей обмена сообщениями поверх уровня связи.

Используйте их, когда:

- вы создаете собственный пользовательский интерфейс чата
- вам необходимо встроить обмен сообщениями на сайт или в приложение
- вы хотите обмениваться сообщениями с клиентами, не раскрывая оператора workspace

## Platform APIs

Platform API предназначены для сценариев администрирования с более высоким уровнем доверия.

Используйте их для:

- предоставление учетных записей и пользователей
- контролируемое администрирование платформы
- управляемые рабочие процессы на уровне облака или партнера

## Как выбрать

1. Если ваша интеграция действует как пользователь workspace, начните с Application API.
2. Если ваша интеграция представляет собой клиент чата для клиентов, используйте Client API.
3. Если ваша интеграция требует подготовки или администрирования на уровне платформы, используйте Platform API.

## Совет по интеграции

- рассматривайте Application API как базовый вариант для большинства бизнес-интеграций
- используйте Client API только для внешнего чата
- используйте Platform API только там, где интеграция действительно требует контроля на уровне платформы

## Если контракт отличается от пользовательского интерфейса

Если конечная точка кажется неясной, сравните ее с сетевыми вызовами, выполняемыми пользовательским интерфейсом One Link Cloud. Пользовательский интерфейс часто является наиболее надежным источником информации о точной структуре запросов и ответов, которая в настоящее время используется в производственных рабочих процессах.
