# FAQ

Здесь собраны короткие ответы на частые вопросы по модели продукта, workspace-структуре, интеграциям и способу настройки платформы под клиента.

## Разделен ли One Link Cloud на отдельные отраслевые продукты?

Нет. One Link Cloud использует одно общее функциональное ядро. Поведение, специфичное для клиента, реализуется посредством конфигурации, доступа, полей, автоматизации, интеграции и настройки AI.

## Должны ли мы создать отдельные workspace для каждой команды?

Только тогда, когда данные или администрирование действительно должны быть разделены. В большинстве случаев одна workspace с несколькими командами и inbox-очереди является лучшей операционной моделью.

## В чем разница между inbox и каналом?

Канал — это технический источник связи. inbox — это операционная очередь, в которой команда управляет этим трафиком.

## Когда следует использовать метки, а не настраиваемые поля?

Используйте метки для быстрой классификации и обработки очередей. Используйте настраиваемые поля или управляемые поля для повторяемых структурированных бизнес-данных.

## Нужен ли нам CRM для использования платформы?

Нет. Команды могут начать только с общения, а структуру CRM добавить позже. CRM становится важным, когда вам нужны воронки, сделки, задачи и более формальное отслеживание бизнеса.

## Нужно ли нам планировать использование платформы?

Нет. Scheduling — это модуль для компаний, которые управляют встречами, услугами и платежами. Это не требуется для рабочих процессов, предназначенных только для обмена сообщениями.

## В чем разница между автоматикой и macros?

Автоматизация запускается автоматически при определенных событиях. Macros запускаются пользователями, когда они хотят быстро выполнить известную последовательность действий.

## Какая поверхность API наиболее безопасна для большинства интеграций?

В большинстве случаев начните с Application API. Используйте Client API только для клиентов чата, ориентированных на клиентов, и Platform API только для потоков подготовки или управляемого администрирования.

## Как внешняя система должна реагировать на изменения в One Link Cloud?

Используйте webhooks или автоматически запускаемые действия webhook. Спроектируйте приемник так, чтобы он был идемпотентным и безопасно обрабатывал повторные попытки.

## Может ли Captain использовать наши собственные знания и инструменты?

Да. Captain можно настроить с помощью помощников, документов, сценариев и пользовательских инструментов на уровне workspace.
