# Fonoster Voice integration contract

This internal page points to the active source-of-truth documents in the Chatwoot repository root.

## Active source of truth

- `ONELINK_FONOSTER_VOICE_CONTRACT.md` - main production contract for Fonoster, `onelink-ai-voice`, and Rails.
- `ONELINK_EXTERNAL_GEMINI_LIVE_VOICEAPP.md` - deployment/runbook for the Onelink-hosted Gemini Live VoiceApp.
- `ONELINK_BRIDGE_API_CONTRACT.md` - bridge command API and inbound route/event callbacks.
- `ONELINK_CRM_GEMINI_SYNC_HANDOFF.md` - short operational handoff/checklist for cross-team production sync.

Older notes in `fonoster-docs/` are archived background. Do not use them as implementation requirements unless the requirement is also present in one of the active documents above.

## Current production split

```text
Fonoster
  telephony execution, PSTN/SIP, media bridge, app refs, recording, technical fallback

onelink-ai-voice
  separate Node service next to Rails, Gemini Live websocket, realtime audio, barge-in, transfer execution

Rails Onelink
  CRM truth, routing policy, customer context, prompts, tools, transcripts, summaries, finalization
```

Rails must not receive realtime audio frames.

## What to send to Fonoster team

Send only:

- private `onelink-ai-voice` gRPC endpoint: `<private-host>:50061`
- optional health endpoint: `<private-host>:8081`
- Fonoster `EXTERNAL` app ref for Onelink AI voice
- network policy for TCP `50061`
- technical fallback behavior
- event/retry/finalize rules from `ONELINK_FONOSTER_VOICE_CONTRACT.md`

Do not send:

- Gemini keys
- Rails internal token
- CRM database access
- prompts
- customer context rules
- CRM tool implementation
- operator selection policy except concrete executable route/transfer targets returned by Onelink

## Rails endpoints

Bridge callbacks:

```text
POST /internal/voice/inbound/route
POST /internal/voice/inbound/event
POST /telephony/internal/events
```

AI runtime endpoints:

```text
POST /internal/voice/ai/context
POST /internal/voice/ai/transcript
POST /internal/voice/ai/tools/:name
POST /internal/voice/ai/control
POST /internal/voice/ai/event
POST /internal/voice/ai/finalize
```

`/event` and `/finalize` are adapter endpoints and must be idempotent.

## Auth summary

Bridge callbacks accept compatible bridge auth headers:

```text
X-Bridge-Secret
X-Telephony-Secret
Authorization: Bearer <token>
```

AI runtime to Rails uses:

```text
Authorization: Bearer <internal voice token>
```

Rails accepts these env aliases for the internal voice token:

```text
VOICE_AGENT_ONELINK_AI_SHARED_SECRET
ONELINK_AI_VOICE_INTERNAL_TOKEN
AI_VOICE_INTERNAL_TOKEN
ONELINK_INTERNAL_SECRET
ONELINK_INTERNAL_TOKEN
```

Use the same value on Rails and `onelink-ai-voice`.

## Required cases

The active contract covers these cases:

1. Inbound AI call through Onelink AI app ref.
2. Outbound AI call through Onelink AI app ref.
3. Operator route.
4. AI transfer to operator.
5. Operator no-answer/busy/failed.
6. Gemini unavailable.
7. Rails context timeout.
8. Voice stream broken.
9. Caller hangup.
10. Duplicate event retry.
11. Idempotent finalize.
12. Recording arriving after finalize.

## Production readiness gate

Before enabling live production AI calls:

1. Fonoster reaches `onelink-ai-voice:50061` privately.
2. Fonoster app endpoint is not local `test-voiceapp`.
3. Fonoster image supports bidirectional `AUDIO_IN` and `AUDIO_OUT`.
4. Gemini keys exist only on the Onelink side.
5. Rails and `onelink-ai-voice` use the same internal token.
6. One inbound AI call passes.
7. One outbound AI call passes.
8. Operator transfer passes.
9. Gemini failure fallback passes.
10. `/event` dedupe and `/finalize` idempotency are verified.
